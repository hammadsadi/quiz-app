Quiz

1 What is JavaScript?

JavaScript is a high-level, interpreted programming language that conforms to the ECMAScript specification. It is used primarily to create and control dynamic website content.

2 What are JavaScript data types?

JavaScript data types include Number, String, Boolean, Object, Undefined, Null, Symbol, and BigInt.

3 What is the difference between "==" and "===" in JavaScript?

"==" checks for equality with type coercion, while "===" checks for equality without type coercion, meaning both value and type must be the same.

4 What is a closure in JavaScript?

A closure is a feature where an inner function has access to the outer (enclosing) function's variables, even after the outer function has closed.

5 Explain "this" keyword in JavaScript.

The "this" keyword refers to the object it belongs to. In a method, it refers to the owner object. Alone, it refers to the global object (window in browsers). In an event, it refers to the element that received the event.

6 What is the difference between "null" and "undefined"?

"undefined" means a variable has been declared but not assigned a value. "null" is an assignment value that represents no value or no object.

7 What is an Immediately Invoked Function Expression (IIFE)?

An IIFE is a function that runs as soon as it is defined. It is defined with the function expression wrapped inside parentheses, followed by another set of parentheses to call it.

8 What is event delegation?

Event delegation is a technique involving the use of a single event listener to manage all events of a particular type for child elements. This is done by exploiting the event bubbling mechanism.

9 What are Promises in JavaScript?

Promises are objects that represent the eventual completion (or failure) of an asynchronous operation and its resulting value. They provide a cleaner way to handle asynchronous operations compared to callbacks.

10 What is the use of the "let" keyword in JavaScript?

The "let" keyword is used to declare variables that are block-scoped, meaning the variables are only accessible within the block where they are defined.
